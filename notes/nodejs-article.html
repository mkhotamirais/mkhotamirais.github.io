Pendahuluan

  • sifat: single thread, assynchronous, non blocking, running v8 engine outside chrome, event driven, highly scalable, cross platform; npm node pakage manager (kumpulan librari yang bisa ditambah ke aplikasi node)
  • fitur utama: file system; buffer; http & https; REPL (read, event, p, loop, console, etc)
  • nodejs (runtime: v8 engine; single thread & asynchronous; berjalan di runtime sendiri; npm) vs php (bahasa pemrograman; zend engine; multi thread & synchronous; apache/iis; conposer)
  • nodejs cocok untuk dynamic single page application (spa); realtime app (chat, miltiplayer); api; treaming app; microservice; command line tool
  • pre requisite: javascript dasar; javascript lanjutan (modern js/es6)
  • system & software requirement: window; code editor; terminal (git bash; cmd)
  • repl (read evaluate print loop)
  • .help; global; .exit; .load
  • code . (membuka vscode lewat bash)

MongoDB

  • diambil dari kata Humongous Database; Humongous artinya besar
  • adalah sebuah database berbasis dokumen yang memiliki skalabilitas dan fleksibilitas yang kita inginkan dan juga memiliki fitur query dan indexing yang kita butuhkan
  • berbeda jenis dengan database sql (mysql, pogradesql, yang ada sqlnya) itu dianggap database relational
  • database berbasis dokumen (salah satu tipe dari NoSQL Database: Not Only SQL)
  • NoSql: shemaless/flexible Schema (schema adalah tanpa mendefinisikan struktur dari database (ddl), struktur tabelnya apa, ada field apa saja ukuran, tipe)
  • Denormalization (tidak normal); non-relational database
  • Unstructured, semi-structured, structured data
  • Tipe NoSQL Databae: key-value store (redis) document based (mongodb, couchdb) column-oriented database (hbase, cassandra) graph database (neo4j)
  • terminologi SQL vs NoSQL
    • tempat penyimpanan sama sama database
    • sql: datanya bisa memiliki tabel; nosql: datanya tidak tabular tapi 'collection' seperti json
    • sql: satu data disebut record/row; nosql: satu data disebut document
    • sql: column/field; nosql: fiels
  • embeded document: field baru yang berbeda dari field sebelumnya
  • mengapa mongodb: nodejs bisa terkoneksi dengan database manapun, alasan mongodb: 1. bentuk data JSON(BSON:binary json)
  • MERN, MEVN, MEAN; react, vue, angular

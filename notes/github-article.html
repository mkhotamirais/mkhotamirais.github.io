  • Git (local) Github (cloud). First open github sigin/sign up -> repository -> fill a new repository -> check readme -> new file [create your content] -> commit -> input message
  • If you want to look the changes visually, go to insights -> network
  • alias graph="git log --all --decorate --oneline --graph".
  • Send your local repo to github: create your local repo with the files -> open github.com -> create new repo -> uncheck readme -> copy (git remote add origin http://github.com/account-name/repo-name.git) -> open git bash -> change directory to your repo -> paste (git remote ...) -> git remote -> git remote -v -> git push -u origin master -> tehn git push
  • Request your github repo to local create your repo on github and some files -> code -> clone -> copy (http...) -> create your local repo -> open terminal/git bash and change directory to your repo -> git clone [paste (http...)] -> git remote (default origin) -> git remote -v (there are two origins: fetch and upush) -> git status -> change the files -> git push
  • if the username and email are requested then: git --list -> git --global (or --local) user.name [yourname]; git --global user.email [youremail] -> git push
  • Fetch / Merge conflict between git local and git cloud if conflict, git push will not work then
  • git fetch (check repo in github) -> git status (check repo in git local) -> graph (there is a branch) -> git pull (resolve the conflict then delete the comment tag) -> git status -> git add . -> git commit -m "message" -> git push

Git

  • Git is one of version control system (VCS) software, the others are Subversion, mercurial, CVS
  • Git originate source code flexibly it's can do revision, calaboration and sharing by recorded snapshot then you can go back to the previous changes
  • download git then "git bash" terminal will be installed automatically
  • Gitbash command: git -v/version, pwd, clear, cd, ls, q
  • open git bash or command prompt create new folder in any path then, change directory to the folder. (these are the commands: git init, git add [newfile], git status, git commit -m "comment", git config, git branch, git help, git log -n; git checkout [5 five digit hash] [file-name]); You should know the three area/state: working tree, staging area and history
  • First: git init (change your folder/directory to a repo/repository, then a folder '.git' will be added to your repo, uncheck "view -> hidden items" to see the ".git". it will record every changes in staging area and history) git status -> git add . (move all inside your repo from working tree to staging area) -> git init -m "message" (move from staging area to history) -> git status;
  • if you add new files you must do: git add . -> git init -m "message"
  • if you change your files you can only do: git commit -am "message"
  • if username adn email are requested then do: "git config --list"/"git --list --show-origin" -> git config --global user.name "your-username" -> git config --global user.email "youremail@gmail.com"
  • git log (see all chages); git log -3 (see the last 3 chages); git log index.txt (see chages in index.txt)
  • git log --oneline --graph; git show hash; git checkout hash
  • git clean -f (delete newfile); git resotre [filename] undo changes or deletions in working tree
  • Git branching: when you init your folder, the file become branch master/main
  • git branch; git branch newbranch; git log; git log --all --decorate --oneline --graph; you can shorten the long git log command to the shorter one by alias
  • alias graph="git log --all --decorate --oneline --graph".
  • git checkout newbranch (moving head(pointer) to newbranch); git branch
  • All you do in the newbrach will not affect to master branch unless both them are merged
  • Git merge if you want to merge newbranch with master branch then the pointer must point to master branch
  • after merging, you can delete the branch that no longer used
  • the conflict occurs if the same files in both branches being edited at the same place

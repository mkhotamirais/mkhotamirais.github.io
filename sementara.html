  • NoSql berbasis dokumen
  • kalau pakai sql merasa lemot, tidak usah langsung pake nosql, belajar dulu optimasi database
  • kita pakai nosql ketika butuh saja. banyak yang tidak butuh tapi ingin pakai
  • ibaratnya nosql itu hp keluaran terbaru yang fancy. kalau hp lama masih berfungsi dan tidak ada masalah, kenapa harus pakai hp baru
  • terkadang yang pakai nosql itu karena gengsi, karena orang lain sudah pakai tapi kita belum
  • nosql (not only sql) selain sql, artinya ada banyak sekali. tiap database nosql punya tugas masing masing lihat{" "} ranking database
  • saat memilih db nosql, jangan sampai salah pilih, kalau sampai salah pilih, kasusnya seperti ke minimarket pakai helikopter
  • ada kasus perusahaan ingin menggunakan elasticsearch, padahal datanya cuma 15 mb, kalau data sekecil itu pakai sql juga bisa bahkan pakai excel juga bisa
  • lalu kapan pakai nosql? karena nosql ada banyak jenisnya maka akan dibahas beberapa saja
  • kapan pakai key-value db, contohnya yang paling populer Redis. db ini hanya berisi key dan value, key seperti primary key dan value itu datanya baik string, json apapun tergantung yang buat. mirip struktur map array, key-nya seperti index di array yang tidak perlu angka. untuk memasukkan data, wajib memasukkan key nya apa, jika duplikat maka akan ditimpa, select data wajib sebutkan keynya, begitupun update dan delete.
  • keuntungan pakai key-value, rata rata key-value db itu menyimpan db nya di memory / ram bukan di hard disk sehingga sepeednya luar biasa
  • kapan pakai document database, contoh paling populer adalah mongodb. document db rata rata menyimpan datanya dalam bentuk json document karena itu sangat fleksibel, tidak butuh join table atau tambah relasi. Tidak ada kolom jadi bisa menambahkan atribut json sesukanya. Tidak bisa join
